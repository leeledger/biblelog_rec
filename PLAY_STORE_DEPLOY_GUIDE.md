## 📌 2. 출시 작업 (언제든지 시작 가능)
### A. 기술적 연결 (상단 주소창 제거)
1. 이미 `assetlinks.json`에 올바른 지문과 패키지명이 적용되어 있습니다. v2.0 배포 시 자동으로 반영됩니다.
2. TWA(Trusted Web Activity)가 해당 파일을 읽어 주소창을 숨기게 됩니다.

### B. 개인정보처리방침 등록
1. 구글 플레이 콘솔 -> '앱 콘텐츠' -> '개인정보처리방침'에 다음 URL을 입력합니다.
2. URL: `https://biblelog.kr/privacy.html`

### C. 앱 패키징 (AAB 파일 생성)
Bubblewrap 도구를 사용하여 배포용 파일을 생성합니다.
1. 터미널에서 다음 명령 실행:
   ```bash
   npx @bubblewrap/cli init --manifest https://biblelog.kr/manifest.webmanifest
   ```
2. 설정 질문에 답하기:
   - Package ID: `com.leeledger.bibleread`
   - Key store 정보: 기존 키가 있다면 경로 지정, 없다면 새로 생성 (비밀번호 안전하게 보관 필수)
3. 빌드 실행:
   ```bash
   npx @bubblewrap/cli build`
   ```
4. 생성된 `app-release-bundle.aab` 파일을 플레이 콘솔에 업로드합니다.

## 📌 3. 필수 준비물 (테스트 및 출시용)
1. **닫힌 테스트 인원:** 최소 12명~20명의 구글 이메일 리스트 (14일간 앱 설치 유지 및 피드백 필요)
   - *최근 구글 정책 변경으로 12명으로 완화된 경우가 있으나, 콘솔 대시보드에 표시된 숫자를 따르세요.*
2. **그래픽 자산:** 
   - 앱 아이콘: 512x512 (투명도 없는 PNG)
   - 그래픽 이미지: 1024x500 (스토어 상단 배너)
   - 스크린샷: 폰(최소 2장), 7인치 태블릿, 10인치 태블릿 각각 준비

### ⚠️ 주의사항
- **웹 서비스 영향 없음:** `.aab` 파일을 빌드하거나 플레이 스토어에 올리는 행위는 현재 `https://biblelog.kr` 사이트 운영에 **아무런 영향을 주지 않습니다.** 
- 앱 버전은 웹의 내용을 단순히 불러오는 방식(TWA)이므로, 웹을 수정하면 앱에도 실시간으로 반영됩니다.
